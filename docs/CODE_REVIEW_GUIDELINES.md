# AI Agent Code Review Guidelines

このディレクトリには、AI エージェントがコードレビューを実行する際のガイドラインが含まれています。

## 📁 ファイル構成

- **`.github/ai-agent-code-review-guidelines.md`** - 全AI ツール共通の総合ガイドライン
- **`.claude/code-review-guidelines.md`** - Claude Code 専用レビューガイドライン  
- **`.codex/code-review-guidelines.md`** - GitHub Copilot CLI 専用レビューガイドライン
- **`.gemini/code-review-guidelines.md`** - Gemini CLI 専用レビューガイドライン

## 🎯 使用方法

### AI ツール別の適用
各AI ツールを使用する際は、該当するガイドラインを参照してください：

1. **Claude Code**: 仕様書駆動開発に特化したレビュー手法
2. **GitHub Copilot CLI**: 対話的・安全性重視のレビュー手法
3. **Gemini CLI**: 多角的分析・探索的レビュー手法

### レビュープロセス
1. **事前準備**: 仕様書（`steering/`, `specs/`）の確認
2. **ガイドライン選択**: 使用するAI ツールに応じたガイドラインを参照
3. **レビュー実行**: ガイドラインに従った段階的レビュー
4. **フィードバック**: 具体的で実行可能な改善提案

## 🔄 継続的改善

これらのガイドラインは、レビューの品質向上と効率化のため定期的に更新されます。
新しいベストプラクティスや改善点があれば、該当するガイドラインファイルを更新してください。

## 📊 品質指標

レビューの効果を測定するため、以下の指標を追跡することを推奨します：
- 発見された問題の重要度別件数
- 改善提案の採用率
- レビューにかかる時間
- コード品質の継続的向上
